## Requirements

### Requirement: Static HTML m3u8 extraction

The system SHALL extract m3u8 playlist URLs from a video page by sending an HTTP GET request to the provided page URL and searching the response HTML body for URLs matching the `.m3u8` pattern.

The system SHALL include the following HTTP headers in the GET request:
- `User-Agent`: A Safari browser user-agent string
- `Referer`: The origin domain of the target page URL

#### Scenario: Single m3u8 URL found in static HTML

- **WHEN** user provides a video page URL
- **AND** the page HTML contains exactly one `.m3u8` URL
- **THEN** the system SHALL return a list containing that single m3u8 URL

#### Scenario: Multiple m3u8 URLs found in static HTML

- **WHEN** user provides a video page URL
- **AND** the page HTML contains multiple `.m3u8` URLs (e.g., different resolutions)
- **THEN** the system SHALL return a list of all distinct m3u8 URLs found

##### Example: Multiple resolution URLs

- **GIVEN** page HTML contains `https://cdn.example.com/video/720p/index.m3u8?token=abc` and `https://cdn.example.com/video/1080p/index.m3u8?token=def`
- **WHEN** the system extracts m3u8 URLs
- **THEN** the system returns both URLs in a list with 2 entries

### Requirement: Dynamic JavaScript m3u8 extraction

When static HTML extraction yields zero m3u8 URLs, the system SHALL fall back to a dynamic extraction mode using a headless WebView browser. The system SHALL load the target page URL in the WebView, intercept all outgoing network requests, and capture any request URL matching the `.m3u8` pattern.

The system SHALL set a maximum page load timeout of 30 seconds for the dynamic extraction mode.

The system SHALL inject JavaScript into the WebView that automatically triggers video playback to force m3u8 network requests. The injected script SHALL attempt to click play buttons using an extensible set of CSS selectors that covers common video player UI patterns, including but not limited to:
- Video.js: `.vjs-big-play-button`, `.video-js .vjs-play-control`
- Generic play buttons: elements with class names containing "play" (case-insensitive)
- DPlayer: `.dplayer-play-icon`
- ArtPlayer: `.art-state`
- Plyr: `.plyr__control`
- Custom poster overlays and play icons that overlay video elements

The system SHALL also support CDN-specific JavaScript extraction snippets registered via the CDN adapter system. Each CDN adapter MAY provide a `js_extraction_snippet` that runs in the WebView context to synthesize m3u8 URLs from page-embedded data (e.g., constructing URLs from tokens, keys, or encoded configuration found in script tags or global variables).

#### Scenario: m3u8 URL generated by JavaScript

- **WHEN** user provides a video page URL
- **AND** static HTML extraction finds zero `.m3u8` URLs
- **THEN** the system SHALL launch a headless WebView to load the page
- **AND** the system SHALL intercept network requests and capture all `.m3u8` URLs

#### Scenario: m3u8 URL loaded after play button click

- **WHEN** the target page requires user interaction (clicking a play button) before loading the m3u8 URL
- **THEN** the injected JavaScript SHALL automatically click play button elements matching the registered CSS selectors
- **AND** the system SHALL capture the resulting `.m3u8` network request

##### Example: motv.app play button triggering

- **GIVEN** the page at `https://imagecdn.motv.app/vodplay/1617-1-1/` uses a Video.js player with a "Play Video" overlay
- **WHEN** the WebView loads the page and the injected script clicks the play button
- **THEN** the system captures `https://motv.multicdn.top/.../042320_001-1pon.m3u8?key=...&time=...`

#### Scenario: CDN adapter provides JS extraction snippet

- **WHEN** a CDN adapter registered in the system provides a non-empty `js_extraction_snippet`
- **THEN** the dynamic capture script SHALL execute that snippet in the WebView context
- **AND** any m3u8 URLs reported by the snippet SHALL be captured alongside URLs found by network interception

#### Scenario: Dynamic extraction timeout

- **WHEN** the headless WebView does not detect any `.m3u8` network request within 30 seconds
- **THEN** the system SHALL abort the extraction
- **AND** return an error indicating that no m3u8 URL was found, prompting the user to manually provide a URL

### Requirement: Resolution label parsing

The system SHALL attempt to parse resolution labels (e.g., 720p, 1080p) from the extracted m3u8 URLs by inspecting URL path segments or query parameters. If resolution cannot be determined, the system SHALL label the entry as "Unknown resolution".

#### Scenario: Resolution identifiable from URL path

- **WHEN** an extracted m3u8 URL contains a path segment like `/720p/` or `/1080p/`
- **THEN** the system SHALL label that URL with the corresponding resolution string

##### Example: Path-based resolution parsing

| URL | Parsed Label |
|-----|-------------|
| `https://cdn.example.com/video/720p/index.m3u8?token=abc` | 720p |
| `https://cdn.example.com/video/1080p/index.m3u8?token=def` | 1080p |
| `https://cdn.example.com/video/stream.m3u8?token=ghi` | Unknown resolution |

#### Scenario: Resolution not identifiable

- **WHEN** an extracted m3u8 URL does not contain any recognizable resolution indicator
- **THEN** the system SHALL label the URL as "Unknown resolution"

---
### Requirement: Token refresh extraction

The system SHALL support re-extracting m3u8 URLs from the same page URL to obtain fresh tokens. The re-extraction MUST use the same extraction strategy (static or dynamic) that succeeded during the initial extraction for that task.

#### Scenario: Re-extraction returns fresh token URL

- **WHEN** the auto-relay engine requests a token refresh
- **THEN** the system SHALL re-fetch the original page URL
- **AND** extract m3u8 URLs using the same strategy as the initial extraction
- **AND** return the m3u8 URL matching the originally selected resolution

---
### Requirement: MotvCDN adapter

The system SHALL include a CDN adapter for `motv.multicdn.top` domains. The adapter SHALL:
- Match URLs containing `motv.multicdn.top` in the hostname
- Support token patching by replacing `key=` and `time=` query parameters from a fresh URL onto an existing URL while preserving the video path
- Detect token expiration by comparing the `time=` parameter value against the current Unix timestamp
- Provide a JavaScript extraction snippet for the WebView that attempts to find the m3u8 URL from motv.app page structures (e.g., parsing video player configuration in script tags, intercepting Video.js source assignments)

#### Scenario: MotvCDN adapter matches URL

- **WHEN** a URL contains `motv.multicdn.top`
- **THEN** the MotvCDN adapter SHALL be selected

##### Example: URL matching

| URL | Matches |
|---|---|
| `https://motv.multicdn.top/JS0271-JS0370/042320_001-1pon.m3u8?key=abc&time=123` | Yes |
| `https://videocdn.avking.xyz/bcdn_token=abc/video.m3u8` | No |
| `https://other.multicdn.top/video.m3u8` | No |

#### Scenario: MotvCDN token patching

- **WHEN** auto-relay obtains a fresh m3u8 URL from motv.multicdn.top
- **AND** the old URL has the same video path but expired key/time parameters
- **THEN** the adapter SHALL replace the `key=` and `time=` values in the old URL with values from the fresh URL

##### Example: Token patching

- **GIVEN** old URL: `https://motv.multicdn.top/JS0271-JS0370/042320_001-1pon.m3u8?key=OLD_KEY&time=1778870000`
- **AND** new URL: `https://motv.multicdn.top/JS0271-JS0370/042320_001-1pon.m3u8?key=NEW_KEY&time=1778880000`
- **WHEN** the adapter patches the URL
- **THEN** result: `https://motv.multicdn.top/JS0271-JS0370/042320_001-1pon.m3u8?key=NEW_KEY&time=1778880000`

#### Scenario: MotvCDN token expiration detection

- **WHEN** a motv.multicdn.top URL has a `time=` value less than the current Unix timestamp
- **THEN** the adapter SHALL report the URL as expired
